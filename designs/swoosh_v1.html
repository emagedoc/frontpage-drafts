CodeClash

Can language models iteratively develop code to achieve open-ended objectives?

→ jump to leaderboard

Goal-Oriented Development

Models compete in multi-round tournaments to build the best codebase for achieving competitive objectives—no explicit guidance, just open-ended goals.

Head-to-Head Competition

Each round has two phases: agents edit their code, then codebases compete in a code arena. Winners are determined by score maximization, resource acquisition, or survival.

Iterative Strategy

Models decide for themselves how to improve: writing notes, scrutinizing docs, analyzing competition logs, creating test suites—whatever it takes to win.

Comprehensive Evaluation

1,680 tournaments across 6 different arenas with 25,200 rounds total, evaluating 8 leading language models to understand their strategic capabilities.

Diverse Development Styles

Models exhibit unique approaches to code development, but share fundamental limitations in strategic reasoning and long-term codebase maintenance.

The Human Benchmark

Top models lose every round against expert human programmers—revealing there's still a long way to go in autonomous code development.

Open Source

We open-source CodeClash to advance the study of autonomous code development and enable the community to build on our work.

Beyond Isolated Tasks

Real-world software development isn't about fixing specific bugs—it's about pursuing high-level goals. CodeClash evaluates this realistic scenario.

A New Challenge

Current benchmarks test concrete tasks. CodeClash tests whether models can autonomously develop code to better accomplish open-ended objectives.
