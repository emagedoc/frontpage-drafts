CodeClash

Language models compete to build winning codebases

We all know that LMs make pretty good junior developers. Give them a small bug or ask them to implement something specific, they'll get it done most of the time.

But the real world has no GitHub issues, only goals.

  • Increase user retention.
  • Cut server costs.
  • Make the app faster.

No one tells you which files to edit or what tests to write. You have to figure that out yourself.

Can language models take an open-ended objective and iteratively develop code to achieve it—without explicit guidance?
CodeClash is a benchmark where language models compete in multi-round tournaments. Each model builds and evolves its own codebase to win head-to-head battles in competitive arenas.

Phase 1: Develop

Write notes, analyze opponent logs, build test suites, refactor code—whatever you think will help.

Phase 2: Compete

Your codebase faces off in the arena. Winners advance. Losers adapt or die.

What we learned:

  • Models have diverse development styles but share fundamental limitations in strategic reasoning
  • Codebases become progressively messy and redundant over time
  • Long-term maintenance is not their strong suit
Top models lose every round against expert human programmers.

We're open-sourcing CodeClash to push the boundaries of autonomous, goal-oriented code development. Because if we want LMs to solve real problems, they need to learn how to think beyond the ticket.
